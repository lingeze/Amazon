# 2025秋计算概论A大作业Amazon棋
本项目是北京大学2025年秋季学期计算概论A课程的大作业。先后实现了Amazon棋的命令行（app分支下）、bot（bot分支下，用于botzone平台的对战）、图形化页面（gui分支下）。
## 命令行
命令行版本主要用于项目前期内部逻辑的测试，但仍然具备了作业要求的所有功能，可以与ai对战，支持游戏中途暂停、退出、存档、读档等功能。在实现该版本时也保证了项目的健壮性，能够根据用户的输入
的规范性做出相应的响应（大概？）。
## BOT
bot本身与botzone平台的交互只需要参考平台文档即可，主要工作在于写AI算法并调试。
### 估价函数
参考了《An evaluation function for the game of amazons》的论文，设计了对当前局面的估价函数。并参考《基于PVS搜索算法的亚马逊棋博弈系统的设计》，对参数做出一些调整。
### Minmax算法与Alpha-beta优化
用Minmax算法进行搜索，并加以Alpha-beta优化，优化搜索效率。
### 迭代加深算法
充分利用时限，每次增大搜索深度。
### 历史得分排序
记录搜索中每个分支的历史得分，搜索前进行排序，优先搜索得分高的。
### PVS零窗口优化
减小Alpha-beta优化的窗口，使得大多数情况能够尽早发生剪枝。
### 置换表优化
用置换表记录已搜索的情况，下次搜到直接返回值。

## GUI
GUI部分，本项目基于SFML库开发。GUI的核心是围绕状态机来构建的，它负责管理游戏从主菜单到对弈、暂停、存档/读档直至游戏结束的完整流程。

程序的核心是一个GameState枚举，它定义了主菜单、游戏对战、暂停、存档/读档和游戏结束等多种状态。主循环根据当前状态执行不同的事件处理和渲染逻辑，界面的各个视觉元素被封装在独立的类中进行绘制，例如 BoardDrawer负责画棋盘，PieceDrawer负责画棋子，而Button类则复用于所有菜单和功能按钮,通过Highlight类向玩家高亮提示可选的棋子和可行的走法。为管理玩家“选子->移动->放障碍”这三步复杂操作，额外设计了Turnstate内部状态机。

本项目利用fstream库，支持对文件进行操作，进行存档读档功能。存档/读档界面提供了10个档位，并实现了覆盖存档前的弹窗确认、读取空档位的错误提示等功能。AI 的走棋过程被特意设置为分步、带有些许延迟的动画，让 AI 的行动看起来更自然、更直观。
